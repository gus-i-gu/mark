# CS & Coding

## Purpose

This project combines Computer Science education, Python learning, software architecture analysis and practical debugging into a single continuous methodology.

Every implementation should reinforce conceptual understanding.

Every debugging session should expose architectural reasoning.

The final objective is the ability to reconstruct software systems from principles rather than memory.

---

# Core Principle

Understand.

Implement.

Validate.

Refactor.

Knowledge always precedes modification.

---

# Kanbans

&&&  Computer Science Fundamentals

General concepts independent of any programming language.

Examples:

- Class
- Object
- Repository Pattern
- Encapsulation
- Composition
- Interfaces

---

&&%

Python Language

Python implementations of Computer Science concepts.

Examples:

- modules
- packages
- self
- __init__
- dataclasses
- decorators
- properties

---

&%%

Project Architecture

Concepts specific to the application's architecture.

Examples:

- RepositoryContract
- ProductService
- Register Workflow
- MainWindow
- Navigation
- Prediction Engine

---

%%%

Libraries and Dependencies

External technologies used by the application.

Examples:

- sqlite3
- pathlib
- typing
- abc
- PySide6

---

&&

Temporary Project Concepts

Concepts discovered during debugging that have not yet received permanent classification.

Once sufficiently understood they migrate into one of the permanent Kanbans.

---

# Permanent Numbering

Each Kanban maintains an independent sequential numbering.

Example

&&&01

&&&02

&&%01

&&%02

&%%01

%%%01

Numbers are never reused.

Concept identifiers remain permanent throughout the project.

# Educational Philosophy

Architecture before syntax.

Definitions before implementation.

Concepts before debugging.

Debugging validates architecture.

Learning follows the software's dependency graph.

The objective is autonomous architectural reasoning.
